# Capstones — NaviOpsEnterprise

The three integrative, portfolio-defining projects that close the curriculum. Each **integrates whole
modules** into a real, self-directed project whose output is a **portfolio package** you can show an
employer. The full briefs live in the lesson folders; this folder holds the **project packages** you
produce.

| # | Capstone | Proves you can… | Role | Brief | Package (you create) |
|---|---|---|---|---|---|
| 34 | **Service Desk** | run a queue — 50 tickets, KBs, escalations, a major incident, a metrics report | Help Desk T1/T2 | [lesson 34](../lessons/34-service-desk-capstone/) | `34-service-desk-project.md` |
| 35 | **IT Support** | own end-user IT — users, permissions, devices, software, docs, incidents | IT Support / Desktop Support | [lesson 35](../lessons/35-it-support-capstone/) | `35-it-support-project.md` |
| 36 | **Junior SysAdmin** | operate infrastructure — provision, accounts, incidents, backups, reports (safely) | Junior SysAdmin / Infra Support | [lesson 36](../lessons/36-junior-sysadmin-capstone/) | `36-junior-sysadmin-project.md` |

## How the capstones build on each other
```
   L34 Service Desk      → work the queue (Modules A–E + H)        → Help Desk T1/T2
   L35 IT Support        → own outcomes  (Modules B,D,F,G + A,E,H) → IT Support / Desktop
   L36 Junior SysAdmin   → own infra     (Modules F,G,H + all)     → Junior SysAdmin / Infra
```
Each is more autonomous and more infrastructure-facing than the last — mirroring the career ladder
(`alignment/ROLE-MAPPING.md`, `SYSADMIN-PATH.md`).

## What makes a capstone "done"
A capstone is complete when its **portfolio package** (in this folder) contains the deliverables listed in
the lesson's §9, the **debrief quiz** is answered to a professional standard, and the work is **sanitized**
(placeholders only — `navi.project.md` HR#1). See `CAPSTONE-GUIDE.md` for how to run and assemble one.
