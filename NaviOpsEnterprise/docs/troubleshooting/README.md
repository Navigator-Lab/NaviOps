# Troubleshooting guides — what you produce here

**This folder starts empty on purpose — you fill it as you complete lessons.** A troubleshooting guide is
the **diagnostic spine** in document form (`navi.project.md` Hard Rule #4):
**Symptoms → Possible Causes → Diagnostic Steps → Resolution Steps → Escalation Criteria →
Post-Incident Documentation.**

## How to create one
1. Copy `docs/templates/troubleshooting-guide.md`.
2. Fill the six-part spine for the lesson's problem class (most-likely cause first; cheap checks first).
3. Save here as `topic.md`, sanitized (placeholders only — HR#1).
