# Access a shared folder

**Applies to:** staff needing access to a team/network shared drive · **KB ID:** KB-0007
**Last reviewed:** 2026-06-21 · **Owner:** Service Desk

## What this covers
How to open a shared folder, what to do if you get "access denied," and how to request access.

## Open a shared folder
- Use the **mapped drive** (e.g. **F:** for Finance) in **This PC**, or type the path into Explorer:
  `\\FS01\Finance`.
- If your mapped drive is **missing**, **restart your computer** (drives map at sign-in) and check again.

## "Access denied" — what it means
Access to shared folders is granted by your **team/role** (a security group), not per person. "Access
denied" usually means one of:
- You're **not in the group** that has access (e.g. you changed teams) → request access (below).
- You were **just added** to the group → **sign out and back in** (access updates at sign-in).
- You can **open but not save** → you have **read-only** access and need **edit** rights → request it.

## Request access (the right way)
Ask your **manager** to approve access to the specific folder, then submit a request to the Service Desk
with: the **folder path** (e.g. `\\FS01\Projects`), the **access level** (read or edit), and **who
approved** it. Access is granted via your team group.

## Still not working?
- You had access "yesterday" and lost it → tell the desk (a team/permission change may have removed you).
- Reference **KB-0007** and include the folder path + what happens (can't open / can't save).
