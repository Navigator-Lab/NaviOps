# `docs/learning/` — what each file is for

The NaviOpsNetwork pedagogy layer. Read this first if you're unsure which doc to open.

| File | What it is | When to read it |
|---|---|---|
| **PROJECT_MISSION.md** | The constitution — *why* NaviOpsNetwork exists, end goal, Definition of Done | once, to understand the mission |
| **CLAUDE_TEACHING_RULES.md** | *How* to teach — the 12-section lesson schema + Integration Lenses A–E (single source of truth) | before writing/grading any lesson |
| **ROADMAP.md** | *What to build & which job it serves* — 36-lesson map + NOC-first career stages | to see the plan / pick the next lesson's role context |
| **JOB_MILESTONES.md** | *When to apply* — application waves, dates, ATS keywords (live job-hunt driver) | when job-hunting / preparing applications |
| **LEARNING_STATE.md** | *Where you are now* — skills learned, next lesson (live progress tracker) | **first, every session** — resume with zero re-explanation |
| **troubleshooting-drills.md** | Shared network failure drills (DNS/DHCP/latency/loss/iface/routing/VLAN/firewall), linked from lessons | when a lesson cross-links a drill, or for interview prep |
| **alignment/** | Job + cert matrices: `NOC-JOB-MATRIX.md`, `CCNA-ALIGNMENT.md`, `NETWORK-PLUS-ALIGNMENT.md`, `PORTFOLIO-STRATEGY.md` | mapping lessons → jobs/certs/portfolio |
| **capstones/** | The three capstone project plans (CCNA / NOC / Network-Security) | at the capstone milestones (34–36) |
| **noc/** | NOC operations modules (alert handling, escalation, ticketing, shift handover, RCA, SLA, outage mgmt) + realistic NOC scenarios | for §6 NOC perspective + NOC capstone |
| **lessons/NN-topic/README.md** | One lesson each (12-section schema output, portfolio artifact) | per lesson |
| **prompts/** | Local-only (gitignored) archive of bootstrap prompts | provenance / history only |

**Two live drivers** to resume from each session: `LEARNING_STATE.md` (pedagogy state) and
`docs/STATUS.md` (project/code state). Everything else is reference or law.
