# IR Workflows

The repeatable incident-response process every lesson and scenario plugs into, plus the capstone
scenario. The *technique* is in the lessons; this is the *process* that turns technique into a
closed, documented incident.

| File | What it is |
|---|---|
| [ir-workflow.md](ir-workflow.md) | the 6-phase workflow (detect → investigate → contain → eradicate → recover → lessons learned) with checklist, evidence-collection, and RCA steps |
| [compromised-server-scenario.md](compromised-server-scenario.md) | the capstone scenario: a benign, self-owned, staged Linux-server compromise to investigate end-to-end |

Report/evidence **templates** live in [`../../templates/`](../../templates/) (incident report,
investigation notes, executive summary, RCA, evidence package, lessons learned).
