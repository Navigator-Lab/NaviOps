# `docs/learning/` — what each file is for

The NaviOps pedagogy layer. Read this first if you're unsure which doc to open.

| File | What it is | When to read it |
|---|---|---|
| **PROJECT_MISSION.md** | The constitution — *why* NaviOps exists, end goal, Definition of Done | once, to understand the mission |
| **CLAUDE_TEACHING_RULES.md** | *How* to teach — the 8-step Gate Rule + Integration Lenses A–E (single source of truth) | before writing/grading any lesson |
| **ROADMAP.md** | *What to build & which job it serves* — numeric lesson map (01–28) + NOC-first career stages | to see the plan / pick the next lesson's role context |
| **JOB_MILESTONES.md** | *When to apply* — application waves (M0–M4), dates, ATS keywords (live job-hunt driver) | when job-hunting / preparing applications |
| **LEARNING_STATE.md** | *Where you are now* — skills learned, next lesson (live progress tracker) | **first, every session** — resume with zero re-explanation |
| **troubleshooting-drills.md** | 7 shared RHCSA-gotcha failure drills, linked from lessons | when a lesson cross-links a drill, or for interview prep |
| **lessons/NN-topic/README.md** | One completed lesson each (Gate Rule output, portfolio artifact) | per lesson |
| **prompts/** | Local-only (gitignored) archive of the bootstrap prompts that originated the system | provenance / history only |

**Two live drivers** to resume from each session: `LEARNING_STATE.md` (pedagogy state) and
`docs/STATUS.md` (project/code state). Everything else is reference or law.

> History: `ROADMAP.md` merged the former `CURRICULUM.md` + `CAREER_STAGES.md` (D15, 2026-06-15)
> to cut doc sprawl — see `docs/DECISIONS.md`.
